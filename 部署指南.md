## 二、部署方案

### 方案一：GitHub Pages（免费）

**优点**：免费、自动部署、集成CI/CD

**步骤**：
1. 将代码推送到 GitHub 仓库
2. 启用 GitHub Pages（设置 > Pages）
3. 配置 source 为 `gh-pages` 分支
4. 首次推送后，Workflow 会自动构建部署

**访问地址**：`https://<username>.github.io/<repo-name>/`

### 方案二：Netlify（推荐）

**优点**：免费套餐、自动部署、CDN加速、SSL证书

**步骤**：
1. 注册 Netlify 账号
2. 连接 GitHub 仓库
3. 配置构建命令：`npm run build`
4. 配置发布目录：`dist`
5. 点击 Deploy

**访问地址**：`https://<site-name>.netlify.app/`

### 方案三：Vercel（推荐）

**优点**：免费套餐、自动部署、CDN加速、边缘计算

**步骤**：
1. 注册 Vercel 账号
2. 连接 GitHub 仓库
3. 自动检测配置并部署

**访问地址**：`https://<repo-name>.vercel.app/`

### 方案四：Nginx 自托管

**优点**：完全可控、灵活配置

**步骤**：
1. 安装 Nginx
2. 配置 nginx.conf：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    root /var/www/maintenance-system/dist;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 安全头
    add_header X-Frame-Options "DENY";
    add_header X-XSS-Protection "1; mode=block";
    add_header Content-Security-Policy "default-src 'self';";
}
```

3. 重启 Nginx：
```bash
sudo systemctl restart nginx
```

### 方案五：Docker 容器部署

**步骤**：
1. 创建 Dockerfile：

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

2. 创建 nginx.conf：

```nginx
server {
    listen 80;
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
```

3. 构建镜像：
```bash
docker build -t maintenance-system .
```

4. 运行容器：
```bash
docker run -p 80:80 maintenance-system
```

## 三、环境变量配置

## 四、域名配置

### 4.1 购买域名
推荐平台：
- 阿里云（国内）：https://wanwang.aliyun.com/
- Namecheap（国际）：https://www.namecheap.com/
- Cloudflare（免费）：https://www.cloudflare.com/

### 4.2 DNS 配置

| 记录类型 | 主机记录 | 记录值 | TTL |
|----------|----------|--------|-----|
| A | @ | 服务器IP地址 | 300 |
| A | www | 服务器IP地址 | 300 |
| CNAME | @ | your-site.netlify.app. | 300 |

### 4.3 SSL 证书

**Let's Encrypt（免费）**：
```bash
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```

**Cloudflare（免费）**：
- 在 Cloudflare 中启用 SSL/TLS（完全模式）
- 自动颁发证书

## 五、安全配置

### 5.1 CSP 配置

在 `index.html` 的 `<head>` 中添加：
```html
<meta http-equiv="Content-Security-Policy" content="
    default-src 'self';
    script-src 'self' 'unsafe-inline';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: https:;
    font-src 'self';
    connect-src 'self' https:;
">
```

### 5.2 安全响应头

在 Nginx 配置中添加：
```nginx
add_header X-Frame-Options "DENY";
add_header X-XSS-Protection "1; mode=block";
add_header X-Content-Type-Options "nosniff";
add_header Referrer-Policy "strict-origin-when-cross-origin";
add_header Content-Security-Policy "default-src 'self';";
```

### 5.3 HTTPS 强制

在 Nginx 配置中添加：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```
